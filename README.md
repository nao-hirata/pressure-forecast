# pressure-forecast

愛犬 Cobu のための気圧予報アプリ。東京の海面更正気圧を1時間ごと・7日先までチェックでき、急な気圧低下をお知らせします。

気圧の変化は人だけでなく犬の体調にも影響すると言われています。Cobu の体調管理のお供にどうぞ。

## 機能

- 海面更正気圧（天気予報の標準・体調管理向き）を1時間ごとに表示
- 最大7日先までの予報を折れ線グラフで表示
- 横スクロールに追従しない固定Y軸
- 急な気圧低下（6時間で約3hPa以上）を自動検出して警告表示
- 現在・6時間後・24時間後の気圧と変化量、7日間の変動幅をひと目で確認
- Cobu の体調不良をグラフから記録し、一覧＋記録日とその前2日（3日分）の気圧グラフで振り返り
- 記録は複数端末で共有（任意・無料の Firebase 設定時。未設定ならその端末内のみに保存）
- iPhone のホーム画面に追加してアプリのように全画面で使用可能（PWA対応）

## 使い方

ブラウザで `index.html` を開くだけです。読み込み時に最新の気圧データを自動取得します。

iPhone のホーム画面に追加する場合:

1. Safari で公開URLを開く
2. 共有ボタン（□に↑）をタップ
3. 「ホーム画面に追加」を選択

## 体調記録

Cobu の具合が悪くなったタイミングを気圧と紐づけて記録できます。

- **記録する**: グラフの下に常に「記録」ボタンがあります。グラフで時刻を選ぶ（長押し→なぞり）とその時刻が、選んでいなければ現在時刻が記録対象になります。ボタンをタップすると日時（編集可）とメモを入力するダイアログが開きます。
- **振り返る**: 「Cobu の体調記録」一覧で記録を新しい順に確認でき、カードをタップすると記録日とその前2日（3日分・記録日が右端）の気圧グラフが開きます。メインのグラフと同じ大きさ・縮尺で表示するので、気圧の下がり方を見比べやすくなっています。記録した日の前後の気圧データは記録自体に保存されるため、しばらく経って気圧の取得範囲（直近7日）から外れた古い記録でも、グラフを振り返れます。
- **削除する**: カードを開き、下部の「🗑 記録を削除」をタップして確認ダイアログで「削除」を選びます。

### 複数端末で共有する（Firebase 設定・無料）

Firebase を設定しない場合、記録は**その端末内（localStorage）のみ**に保存されます。複数端末で共有したい場合は、無料の Firebase Firestore を使います（Spark プラン・クレジットカード不要）。

1. [Firebase コンソール](https://console.firebase.google.com/) でプロジェクトを作成
2. 「Firestore Database」を作成
3. プロジェクト設定でウェブアプリを登録し、`firebaseConfig` の値を取得
4. `index.html` 冒頭の `firebaseConfig` に貼り付け（`YOUR_...` を実際の値に置き換え）
5. Firestore のセキュリティルールを設定:

   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /cobu_logs/{doc} { allow read, write: if true; }
     }
   }
   ```

### 不正な書き込みを防ぐ（App Check・任意）

Firestore のルールだけでは、サイトの URL を知った第三者や、`apiKey` を抜き取って他サイト・直接 API から接続する者を防げません。これを防ぐには App Check（reCAPTCHA v3・無料）を設定します。reCAPTCHA v3 は裏で動くスコア方式で、**利用者に画像認証は表示されず操作の手間もありません**。

1. [reCAPTCHA 管理画面](https://www.google.com/recaptcha/admin/create) で「reCAPTCHA v3（スコアベース）」のキーを作成。ドメインに GitHub Pages のホスト名（例 `<ユーザー名>.github.io`）と、ローカル確認用の `localhost` を登録 →**サイトキー**と**シークレットキー**が発行される
2. Firebase コンソール →「App Check」→ ウェブアプリを **reCAPTCHA v3** で登録し、**シークレットキー**を貼り付ける（シークレットキーはコンソールのみ。コードには絶対に入れない）
3. **サイトキー**を `index.html` 冒頭の `RECAPTCHA_SITE_KEY` に設定（サイトキーは公開情報なのでコードに載せて問題ない）
4. まず enforce せずにサイトを開き、App Check 画面のメトリクスで検証済みリクエストが記録されることを確認
5. 問題なければ App Check で Cloud Firestore の **「強制（Enforce）」を有効化**。以降、App Check トークンの無いアクセスは拒否される

> **App Check の効果と限界**: 「あなたのサイト経由のアクセスのみ許可」する仕組みで、他サイトからの `apiKey` 悪用を防ぎます。ただし**人は区別しない**ため、サイトの URL を開いた第三者は利用できます。特定の人だけに限定したい場合は、別途ログイン認証（例: Google ログイン）とアカウント単位のルールが必要です。

> **セキュリティ上の注意**: 上記ルールは「`cobu_logs` を誰でも読み書きできる」設定です。`apiKey` がコードに含まれるのは Firebase の仕様上問題ありません。保存するのは犬の体調メモで機微性が低いため、URL を身内だけで共有する個人利用では現実的です。

## データソース

気圧データは [Open-Meteo](https://open-meteo.com/) から取得しています（[CC BY 4.0](https://creativecommons.org/licenses/by/4.0/)）。APIキーは不要です。

地点は東京（緯度 35.68 / 経度 139.69）に固定しています。`index.html` 内の `LAT` と `LON` を書き換えれば他の地点にも変更できます。

## 技術
